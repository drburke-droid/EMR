
OPTOMETRY CONTEXTUAL FINDING TREES SPEC
Developer-ready menu structure for touch-based charting MVP

Purpose
This file defines anatomy regions, contextual finding trees, diagnosis lists, and treatment plan blocks for a touch-based optometry charting system that generates text for copy-paste into an existing EMR.

Scope
This version is focused on:
Symptoms
History
Clinical findings
Diagnosis selection
Treatment plan selection
Text output generation

This version does NOT include:
AI
Billing
Scheduling
Full EMR replacement

==================================================
1. CORE UI MODEL
==================================================

Workflow
1. User selects exam module
2. User taps anatomy region
3. Contextual menu appears with likely findings for that region
4. User taps finding
5. Successive qualifier menus appear
6. System generates structured text
7. User repeats on same or different anatomy
8. User selects diagnoses
9. User selects treatment plan blocks
10. Final text output is copied into EMR

Menu design rules
Only show likely findings for the selected anatomy
Show most common findings first
Keep each branch shallow where possible
Allow "More" for uncommon findings
Allow repeat finding entry in same structure
Allow copy to fellow eye
Allow exact size, degree, or free text only when needed

Output sections
History
Symptoms
Findings
Diagnosis
Plan
